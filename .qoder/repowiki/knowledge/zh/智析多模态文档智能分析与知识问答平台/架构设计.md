- **表现层**：`src/app.py` 作为 Streamlit 入口，通过 `st.session_state` 管理多步骤工作流状态（解析→分析→图谱→问答）。
- **核心逻辑层**：采用模块化设计，`doc_parser` (PyMuPDF/pdfplumber) 负责多模态提取；`nlp_pipeline` (Transformers/KeyBERT) 执行实体识别与摘要；`knowledge_graph` (NetworkX) 构建实体关系网；`rag_engine` (LangChain/ChromaDB) 实现向量检索与 LLM 生成。
- **数据流向**：文档经解析切块后，一方面流入 NLP/KG 模块进行结构化挖掘，另一方面流入 RAG 引擎建立向量索引，支持基于上下文的智能问答。
- **依赖管理**：通过 `requirements.txt` 统一管理数据科学、CV、NLP 及 LLM 相关依赖，支持 OpenAI API 与本地 Ollama 双模式切换。
